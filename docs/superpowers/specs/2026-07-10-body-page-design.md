# 身体页面设计

## 概述
重写身体数据页面，从当前简约布局升级为完整分析视图。修复 `body_latest()` bug（API 返回倒序导致最新日期显示为 04-11），新增双轴折线图、阶段统计、变化量指标、BMI 和数据表格。

## 发现的问题
- 训记 API 返回的 records 按**倒序排列**（最新在前）
- `body_latest()` 遍历覆盖全部记录，最后一条（最旧）覆盖了最新值 → 最新体重日期显示 04-11
- 修复方案：改用 API 返回的 `res.latest` 字段，或取第一条（最新）记录

## 后端改动

### analysis.py
- **修复 `body_latest()`** — 改为取第一条记录（最新）而不是覆盖到最后，或直接返回 API 的 `res.latest`
- **新增 `body_stats(records)`** — 统计阶段内最高/最低/平均体重和体脂率
- **新增 `body_changes(records)`** — 计算每条记录较上一次同类型的变化量（weight_diff, bodyfat_diff）
- **新增 `calculate_bmi(weight_kg, height_cm)`** — BMI = weight / (height/100)²
  - 身高从配置或首个 body 记录获取，暂设占位，用户可自行修改

### routes/body.py
- 传给模板的上下文新增：
  - `body_stats` — 期间统计
  - `changes` — 各记录的变化量（用于表格列和卡片箭头）
  - `bmi` — 当前 BMI 值
  - `range_days` — 当前选中的时间范围（默认 90）
- 支持 `?range=30|90|180|all` 查询参数切换日期范围

### api_client.py / data_service.py
- 无需改动，现有 `get_body()` 已支持全部所需数据

## 前端组件

### CSS（static/css/app.css）
当前 `app.css` 已有 `.grid-2` `.grid-3` `.card` `.stat` `.stat-value` `.stat-label` `.chart-container` 类。新增：
- `.grid-4` — 4 列网格用于顶部的 4 张统计卡
- `.dual-axis-chart` — 全宽双轴图容器
- `.stats-2col` — 两列统计面板
- `.data-table` / `.data-table th` / `.data-table td` — 表格样式
- `.change-up` / `.change-down` — 趋势颜色（红涨绿跌/绿涨红跌）

### body.html（重写）
- **标题行** — "身体数据" + 日期范围按钮组（1月/3月/半年/全部）
- **4 统计卡片** — 用 `.grid-4` 布局
  - 当前体重（值 + 较上次变化箭头）
  - 当前体脂率（值 + 较上次变化箭头）
  - BMI（值）
  - 期间变化（体重变化 + 体脂率变化）
- **双轴折线图** — Chart.js 双 Y 轴：体重（kg, 蓝色实线）+ 体脂率（%, 红色虚线）
  - 鼠标悬停显示日期 + 两个数值
  - 响应式 + maintainAspectRatio: false
- **两列统计面板** — 体重统计 / 体脂率统计，各显示最高/最低/平均
- **历史记录表格** — 可滚动，显示每条记录的日期、体重、体脂率、BMI、较上次变化
  - 变化值带方向箭头和颜色
  - 同一日期的体重和体脂率合并在同一行
- **日期范围切换** — 通过 URL 查询参数 `?range=` 重新请求

## 技术细节

- Chart.js 双轴：`yAxisID` 属性，左侧 weight Y 轴 `beginAtZero: false`，右侧 bodyfat Y 轴
- 数据合并：同一日期可能有多个同类记录（如同一天多次称重），取该日平均值
- 双轴图数据源：从 `weight_trend` 和 `bodyfat_trend` 分别取，按日期对齐
- 变化量计算：遍历排序后的 records，每种 type 分别与前一条比较（非连续日期也算）
- 表格按日期降序排列，方便看最新记录在顶部
- 初始默认 3 个月；切换范围通过重新请求路由（加 `?range=` 参数）
