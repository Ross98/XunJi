
# 训记数据分析工具 设计文档

## 概述

基于训记官方 Open API（训练/饮食/身体三条 skill）的数据分析 Web 工具。目标：
整合训练、饮食、身体数据，提供历史回顾、趋势分析、训练规划能力。

## 技术栈

- **后端**: Python FastAPI
- **模板引擎**: Jinja2（服务端渲染）
- **交互**: HTMX（无刷页面更新）
- **图表**: Chart.js（浏览器端渲染）
- **数据缓存**: SQLite
- **数据分析**: pandas
- **部署**: Docker + uvicorn + nginx（单容器方案）

理由：无前端构建链，单进程可运行，pandas 天然适合数据分析场景，部署成本最低。

## 页面结构

| 路径 | 页面 | 功能 |
|------|------|------|
| `/` | 总览仪表盘 | 最近训练摘要、体重趋势小图、今日热量概览 |
| `/training` | 训练回顾 | 日历/列表切换，点某天看详细训练内容，趋势图 |
| `/diet` | 饮食记录 | 按日期查看饮食，营养比例统计 |
| `/body` | 身体数据 | 体重/体脂趋势折线，围度变化 |
| `/plan` | 训练规划 | 生成训练安排，预览后写回训记 |
| `/import` | CSV 导入 | 上传 CSV 导入历史数据 |

## 架构

```
浏览器 ←→ FastAPI + Jinja2 + HTMX
                │
         ┌──────┴──────┐
         │ 分析引擎     │
         │ (pandas)     │
         └──────┬──────┘
                │
         ┌──────┴──────┐
         │ 数据层       │
         ├─ SQLite 缓存 │
         ├─ 训记 Open API
         └─ CSV 解析    │
```

## 数据流（策略 C）

1. 用户打开页面
2. 优先读 SQLite 缓存，秒开展示
3. 后台静默请求训记 API 拉最新数据
4. 有新数据 → HTMX 推送通知条「有更新，点此刷新」
5. 无新数据 → 静默，不打扰用户
6. 用户点「刷新」→ 更新 SQLite 缓存

缓存 key: `{skill_type}:{query_params}`，每次拉取打时间戳。

## API 调用层

每个 skill 对应一个 API 客户端模块：

- `api/training.py` — 读/写训练记录
- `api/diet.py` — 查询/搜索/写回饮食
- `api/body.py` — 查询/写入身体数据

所有请求统一处理：
- `--compressed` gzip 解码
- 鉴权头注入
- 限频自动等待（`retry_after_ms`）
- 错误分类（`apikey invalid` / `仅VIP可用` / `too frequent`）

## 分析功能（初版）

### 训练分析
- 周/月/年训练容量趋势（组数 × 重量 × 次数）
- 各部位训练频率分布
- 动作进步追踪（卧推/深蹲等关键动作重量曲线）
- 训练耗时、RPE 趋势

### 饮食分析
- 日/周热量摄入趋势
- 蛋白/碳水/脂肪比例分布

### 身体分析
- 体重/体脂率趋势线
- 围度变化追踪

### 跨 skill 联动
- 热量差 = 摄入 - 消耗
- 训练周期 vs 体重变化关联

## 写回流程

写回操作（训练/饮食记录）统一走安全流程：

1. Agent 生成结构数据
2. 调用 API `dry_run: true` 校验
3. 展示变更摘要给用户
4. 用户确认
5. 调用 API `dry_run: false` 真正写回
6. 更新缓存

## 规划模块

训练规划生成器（`/plan`）：
- 基于历史训练数据（容量、频率、恢复周期）推荐下次训练
- 输出结构化训练内容
- 预览后走写回流程写入训记

## 安全与隐私

- API Key 只在服务端使用，不传到浏览器
- SQLite 缓存包含个人训练数据，注意文件权限
- `.env` 文件列入 `.gitignore`

## 部署方案

```dockerfile
FROM python:3.11-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install -r requirements.txt
COPY . .
CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
```

单容器，挂载 SQLite 数据卷持久化缓存。

## 非功能性约束

- 页面响应 < 500ms（缓存命中时）
- 后台刷新不阻塞页面渲染
- 限频冲突时友好提示，不报错
- 支持离线查看缓存数据
