# 训练页面重新设计

## 背景

当前训练页面只显示当天的训练数据，功能单一。需要扩展为历史回顾 + 训练分析的综合页面。

## 布局：上下分块

### 上半部分 — 日历 + 当天详情

- **日历**：月视图，点某天通过 HTMX 加载当天训练详情（无需刷全页）
- **有训练记录的日子**显示小圆点标记
- **左右箭头**切换月份（当前只拉取已缓存月份的数据）
- **当天详情卡片**：训练数/总组数/总容量摘要 + 各训练的动作组列表

### 下半部分 — 4 个分析标签页

| 标签 | 说明 | 图表 |
|------|------|------|
| 容量趋势 | 最近 N 天训练容量折线（默认 30 天，可选 7/30/90） | Chart.js 折线图 |
| 部位分布 | 近 30 天推/拉/腿各部位训练频次占比 | Chart.js 饼图 |
| 动作追踪 | 下拉选一个动作，看该动作重量/组数历史变化 | Chart.js 折线图 |
| 频率日历 | 近 90 天训练热力图（GitHub 贡献图风格） | CSS 网格热力图 |

## 数据流

- 页面加载 → 从缓存读取所有训练日期 → 渲染日历标记
- 点某天 → HTMX 请求 `/training/{date}` → 返回 HTML 片段 → 替换详情区域
- 日历上下月 → HTMX 重新加载日历片段
- 分析图表的原始数据来源于 SQLite 缓存，不走 API

## 技术实现

- `routes/training.py`：重构为多个端点（`GET /training`、`GET /training/{date}`、`GET /training/calendar/{year}/{month}`）
- `templates/training.html`：重写为日历 + 详情 + 图表布局
- `templates/training_detail.html`：当天训练详情片段（HTMX target）
- `templates/training_calendar.html`：月历片段（HTMX target）
- `analysis.py`：补充部位分布分析函数
- 热力图：纯 CSS + HTML 实现，无需额外库
- `static/css/app.css`：补充日历和热力图样式

## 后续可扩展

- 周期化训练分析（每 4 周对比）
- 疲劳管理（容量 / RPE / 频率计算恢复建议）
